# Budget

We know there may be unexpected charges for things we are running, so it'll be a good idea to set a Budget.

Here, I've set a budget of $20 dollars. We'll set our start and expiration as well. 

This can be very handy to maintain costs, especially in smaller or more budget oriented organizations.

<img width="2016" height="989" alt="image" src="https://github.com/user-attachments/assets/5cf9d041-e409-47c4-a01a-dee1acf5713f" />

Next, we will set alerts. 

We want to be informed when are actual cost or forecasted costs goes over our set limit.

<img width="1997" height="928" alt="image" src="https://github.com/user-attachments/assets/8a50d532-fd8b-42f1-9a96-78e2f1b6be42" />

You can specify when your alert triggers and who it alerts

**It is important to note that setting a budget will NOT stop when the budget is reached, it will only alert you.**
